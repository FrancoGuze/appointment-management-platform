# Changes Made on 2026-03-05

## Summary

Today we moved the project from basic booking CRUD to role-aware, signed-session authorization with admin and professional operational flows.

---

## 1) Authentication and Session Security

- Implemented password verification using `scrypt` + `timingSafeEqual` in `src/services/users.ts`.
- Added functional login endpoint:
  - `POST /api/users/login`
  - Validates credentials and updates `last_login`.
- Added logout endpoint:
  - `POST /api/users/logout`
- Introduced signed session cookies:
  - `src/lib/auth-session.ts`
  - HMAC SHA-256 token signature
  - Embedded expiration (`exp`)
  - Embedded `userId` and `role`
- Removed insecure fallback to `SUPABASE_SECRET` for session signing.
  - Session signing now requires `AUTH_SESSION_SECRET`.

---

## 2) Route Protection and Role-Based Access

- Replaced deprecated middleware convention with `proxy.ts`.
- Added protected route guard via signed cookie validation:
  - `/admin/*` requires `admin`
  - `/professional/*` requires `professional` or `admin`
- Redirect behavior:
  - Unauthenticated -> `/user?redirect=...`
  - Forbidden role -> `/user?error=forbidden`

---

## 3) Users and Roles

- Added user login service functions and role-aware user retrieval.
- Added admin-only users API:
  - `GET /api/users` -> list users
  - `PATCH /api/users` -> update role
- Added role update audit log events.
- Aligned app role model with DB enum values:
  - Changed from `user` to `client`
  - Active role set: `client | professional | admin`

---

## 4) Appointment Authorization and Behavior

- Hardened `POST /api/appointments`:
  - `userId` no longer accepted from client body.
  - `userId` resolved from validated signed session token.
  - Unauthorized requests return `401`.
- Added role-scoped `GET /api/appointments`:
  - `admin` -> all appointments
  - `professional` -> only assigned appointments
  - `client` -> only own appointments
- Extended `PATCH /api/appointments` for:
  - Status updates (`scheduled`, `completed`, `cancelled`, `no_show`) with role checks
  - Admin-only professional assignment (`professionalId` set/unset)
- Added booking audit logs:
  - `booking_created`
  - `booking_status_updated`
  - `booking_professional_assigned`

---

## 5) Professional Assignment Support

- Added DB migration file:
  - `database/migrations/2026-03-05_add_professional_assignment.sql`
- Migration adds:
